# Blog Publishing Checklist

Pre-publication checklist for the CUDL blog post. Track items as you work through them.

## Choose Your Blog Version

Three drafts are provided — pick one (or let the blog team decide):

| File | Type | Best for |
|---|---|---|
| `draft-blog-post.md` | Pure technical walkthrough | AWS Big Data Blog — readers who want to build it themselves |
| `draft-blog-customer-story.md` | Pure customer success story | AWS Public Sector Blog — Rize CU narrative with light technical detail |
| `draft-blog-combined.md` | Combined story + walkthrough | **Recommended** — Rize CU narrative framing with full technical depth |

- [ ] Decide which version to publish (recommend: combined)
- [ ] Get Rize CU approval on their name, quotes, and story framing
- [ ] Fill in actual Rize CU spokesperson name and title in quote placeholders
- [ ] Confirm Rize CU is comfortable with the CDFI / financial inclusion framing

## Blog Post Content

- [ ] Replace `[Author Name 1]` and `[Author Name 2]` with real names and bios
- [ ] Replace `<your-org>` GitHub URLs with the actual public repo URL
- [ ] Render the architecture diagram (drawio or Mermaid) and insert into the blog post
- [ ] Verify all code snippets compile/run correctly
- [ ] Verify cost estimates are current (check AWS pricing pages)
- [ ] Get peer review from at least one other SA/engineer
- [ ] Get editorial review (AWS blog team has style guidelines)
- [ ] Confirm the blog post passes the AWS blog legal review
- [ ] Get Rize CU sign-off on final published version

## Repository Prep (Before Publishing)

## Security Review

- [ ] Run `cdk synth` and review generated CloudFormation for any secrets/PII
- [ ] Confirm no AWS account IDs are hardcoded anywhere
- [ ] Confirm no IAM credentials or API keys in the codebase
- [ ] Confirm all S3 buckets block public access
- [ ] Confirm all S3 buckets enforce SSL
- [ ] Confirm RDS is in isolated subnets
- [ ] Confirm KMS key rotation is enabled
- [ ] Review IAM policies for overly broad `Resource: *` statements
- [ ] Run git log to confirm no sensitive data in commit history
- [ ] Confirm .gitignore excludes cdk.context.json (contains account IDs)

- [ ] Include architecture diagram as a separate high-res image file
- [ ] Include link to the public GitHub repo
- [ ] Tag with: AWS Glue, AWS CDK, Step Functions, Data Lake, Financial Services
- [ ] Suggest publication date (coordinate with Rize for their announcement)
